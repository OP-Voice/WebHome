## Research Workflow

### Phase 1: Planning
1. **Define Research Question**: What specific issue are we investigating?
2. **Identify Data Sources**: What data do we need and where can we get it?
3. **Choose Visualizations**: What charts/graphics will best tell the story?
4. **Determine Scope**: Standalone page, modal component, or both?

### Phase 2: Development  
1. **Copy Template**: Use `_template.html` as starting point
2. **Gather Data**: Collect and organize data in `/data/` folder
3. **Create Charts**: Implement interactive visualizations
4. **Write Content**: Develop narrative around the data
5. **Test Integration**: Ensure links work from main pages

### Phase 3: Integration
1. **Add Navigation**: Update shared components if needed
2. **Cross-Reference**: Add links from relevant main pages  
3. **Create Modal**: If content warrants preview functionality
4. **Document**: Update this README with new topic status

## Content Guidelines

### Executive Summary
- Keep to 2-3 key findings and recommendations
- Use bullet points for scannability
- Include specific numbers/percentages when possible

### Data Visualization
- Always include data sources and methodology
- Use consistent color scheme (brand colors)
- Ensure charts are mobile-responsive
- Provide alt text for accessibility

### Implementation Roadmap
- Break into clear phases with timelines
- Include specific action items
- Identify stakeholders and decision points
- Connect to existing city processes/calendars

## Quality Checklist

Before publishing new research:

- [ ] Data sources clearly documented
- [ ] Charts display correctly on mobile
- [ ] Links work from main site pages
- [ ] Executive summary under 200 words
- [ ] Implementation timeline is realistic
- [ ] Content follows brand voice/tone
- [ ] Page loads in under 3 seconds
- [ ] Navigation integration works
- [ ] Shared components load properly
